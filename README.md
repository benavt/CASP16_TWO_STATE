# CASP Data Processing

This repository contains scripts and utilities for processing, analyzing, and visualizing data for the CASP (Critical Assessment of Structure Prediction) assessment. The workflow includes processing scores, identifying missing data points, and generating plots and summary files.

[![zenodo 17247592](https://github.com/user-attachments/assets/3a87848a-d955-44a5-a79a-02e692a15437)](https://doi.org/10.5281/zenodo.17247592)

This work is used in the following publication:

Dube, N., Ramelot, T. A., Benavides, T. L., Huang, Y. J., Moult, J., Kryshtafovych, A., & Montelione, G. T. (2025). Modeling Alternative Conformational States in CASP16. bioRxiv. https://doi.org/10.1101/2025.09.02.673835

## Installation

### Option 1: Using requirements.txt (Recommended)
Install all dependencies using the provided requirements file:
```bash
pip install -r requirements.txt
```

### Option 2: Manual Installation
Install dependencies individually:
```bash
pip install pandas matplotlib adjustText tqdm
```

### Option 3: Using a Virtual Environment (Recommended for Development)
Create and activate a virtual environment to avoid conflicts:
```bash
# Create virtual environment
python -m venv casp_env

# Activate virtual environment
# On Windows:
casp_env\Scripts\activate
# On macOS/Linux:
source casp_env/bin/activate

# Install dependencies
pip install -r requirements.txt

# Deactivate when done
deactivate
```

## Quick Start

1. **Clone or download this repository**
2. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```
3. **Ensure your data files are in the `data/` directory**
4. **Run the main processing script:**

5. **Check the `output/` directory for generated plots, CSVs, and tables**

For a complete workflow, see the [Complete Workflow](#complete-workflow) section below.

## Script Overview and Usage

### 1. `process_two_state_score.py`
**Purpose**: Processes the gathered data to compute two-state scores for each target and score type, generates summary CSVs in `output/OUTPUT_CSVS/`, and creates scatter and bar plots in `output/PLOTS/`.

**Usage:**
```bash
python scripts/process_two_state_score.py
```

**What it does:**
- Automatically processes all targets and score types defined in `TARGET_SCORE_DICT`
- Computes two-state scores by combining the best scores from different model/version combinations
- Generates scatter plots comparing scores from V1 and V2 reference states
- Creates stacked bar charts showing combined scores
- Saves detailed CSV outputs with all scoring information
- Handles special cases for different targets (M1228, M1239, R1203, T1214, T1228, T1239, T1249)

**Targets and Score Types Processed:**
- **M1228**: BestDockQ, GDT_TS, GlobDockQ, GlobalLDDT, TMscore
- **M1239**: BestDockQ, GDT_TS, GlobDockQ, GlobalLDDT, TMscore
- **R1203**: GDT_TS, GlobalLDDT, Composite_Score_4, TMscore
- **T1214**: GDT_TS, GlobalLDDT, TMscore, Composite_Score_4
- **T1228**: GDT_TS, GlobalLDDT, TMscore
- **T1239**: GDT_TS, GlobalLDDT, TMscore
- **T1249**: AvgDockQ, GlobalLDDT, GDT_TS, TMscore

### 11. `original_pipeline_by_NamitaDube_2024/` (Legacy Directory)
**Purpose**: Contains the original modular pipeline for computing Composite Score 4 (CS4) across predicted protein structures. This is legacy code that does not function in the current environment but served as the foundation for developing the current repository's scripts.

**Author**: Namita Dube (2024)  
**Status**: Non-functional legacy code, preserved for reference

**What it contains:**
- **Modular pipeline design**: Extract → merge → normalize → score → plot workflow
- **10 Python scripts** for processing LGA outputs, GlobalLDDT logs, and local GDT/LDDT tables
- **Makefile** for automated pipeline execution
- **Composite Score 4 calculation**: Mean of 5 components (Local_LDDT, Local_GDT, binary thresholds for Global_LDDT>80, Global_GDT>80, and g_RMS)
- **Configurable thresholds** for Global_GDT and Global_LDDT (default 80)
- **Plotting capabilities** for bar plots and scatter diagnostic plots

**Key Scripts:**
- `extract_global_gdt_rms.py` - Parse Global GDT_TS and g_RMS from LGA outputs
- `extract_global_lddt.py` - Parse Global LDDT from .lddt outputs  
- `extract_local_gdt.py` - Compute Local GDT averages for residue ranges
- `extract_local_lddt.py` - Compute Local LDDT averages for residue ranges
- `merge_all.py` - Merge metrics into combined_results.csv
- `normalize_metrics.py` - Apply sigmoid scaling and ×100 scaling
- `compositescores.py` - Compute Composite_Score_4 (CS1-CS4)
- `plot_models_bar.py` - Generate bar plots
- `plot_best_per_group.py` - Plot best scores per group
- `plot_scatter_diagnostic.py` - Generate scatter diagnostic plots

**Note**: This directory is preserved for historical reference and to understand the evolution of the current processing pipeline.

## Output Files

### CSV Outputs (`output/OUTPUT_CSVS/` directory)
Each output CSV file contains detailed scoring information with columns:
- **Group**: The submission group identifier (e.g., TS314)
- **Group_Name**: Human-readable group name
- **Combined_Score**: The sum of the two best scores for the group
- **Best_v1_ref**: The best score using a v1 reference
- **Best_v2_ref**: The best score using a v2 reference
- **V1_Model_For_Combined_Score**: Model identifier for the V1 score
- **V2_Model_For_Combined_Score**: Model identifier for the V2 score
- **Best_Score**: The highest score among all combinations
- **Best_Source**: Which combination produced the best score
- **v1_v1_Score, v1_v2_Score, v2_v1_Score, v2_v2_Score**: Individual scores for each combination
- **v1_v1_ModelNumber, v1_v2_ModelNumber, v2_v1_ModelNumber, v2_v2_ModelNumber**: Model numbers for each score

### Plots (`output/PLOTS/` directory)
- **Scatter plots**: Comparing V1 vs V2 reference states
- **Stacked bar charts**: Horizontal and vertical versions with and without special highlighting
- **Special features**: AF3 baseline highlighting, inset zoom regions for specific targets
- **Full axis plots**: Enhanced scatter plots with complete axis ranges
- **Simple bar plots**: Clean, minimal bar chart visualizations
- **Multi-panel plots**: Combined TM and GDT score visualizations
